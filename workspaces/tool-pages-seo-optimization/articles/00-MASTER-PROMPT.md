# Master Prompt for 2000-Word Tool Articles

## Context

You are writing professional, SEO-optimized articles for a free tool website. Each article is 2000+ words and serves multiple purposes:
1. Help real users solve problems with the tool (user value first)
2. Rank on Google for target keywords (SEO)
3. Drive traffic to the tool

## Quality Standards (CRITICAL)

- **Human-written tone**: Natural, conversational, authentic. NOT AI-generated sounding (no generic phrases like "In this article, we will explore...").
- **Genuinely valuable**: Solve real problems. Include concrete examples. Give practical advice readers can use immediately.
- **SEO-optimized without penalties**: Keywords naturally woven in. NO keyword stuffing. Reads naturally. NO black-hat tactics.
- **Original content**: Not plagiarized. Not duplicated from other websites.
- **Well-structured**: Clear headings, short paragraphs (3-4 sentences max), good scannability.
- **Engaging**: Keeps readers interested. Builds trust. Action-oriented.

## Article Structure (2000+ words)

### 1. Introduction (200 words)
- Hook the reader with a relevant problem or relatable scenario
- Problem statement: What problem does the tool solve?
- Quick explanation: What is the tool?
- Why it matters: Benefit summary
- Lead-in: "Let's dive into how to use it"

**Example hook**: "Writing effective email subject lines is harder than it looks. You spend time crafting the perfect subject, hit send... and crickets. Most of your emails land in spam or get ignored."

### 2. How to Use (500 words)
- Step 1-5: Numbered, clear steps with real example data
- Each step should have context: what you're doing and why
- Include at least one concrete walkthrough with real input/output
- Mix in common use cases naturally within steps
- End with: "That's the basic workflow. Now let's look at what makes this tool powerful."

**Example**: For JSON Formatter, show steps like:
1. Paste your JSON into the input box on the left
2. Click the "Format" button to beautify and validate
3. Check the output — nicely formatted with indentation
4. Use "Copy" to copy the formatted result
5. Optional: Click "Minify" if you need production-ready code

### 3. Features & Benefits (400 words)
- 3-4 key features: What can the tool do?
- For each feature: What it does + Why it matters
- Who benefits: Different user types (developers, marketers, freelancers, etc.)
- Real impact: Time saved, quality improved, etc.
- Why this tool: Advantages vs manual alternatives (without naming competitors)

**Example**: For Email Subject Tester:
- Feature 1: Preview across 20+ email clients → Prevents subject line cutoff
- Feature 2: Spam word detection → Avoids spam filters
- Feature 3: Character counter → Ensures text fits on all devices

### 4. Examples & Use Cases (500 words)
- 3-4 real-world scenarios with concrete outcomes
- Each scenario: Situation → How they used tool → Result
- Different industries/user types (developer, marketer, freelancer, student, etc.)
- Make scenarios achievable (not fantasy, not too basic)
- Show concrete numbers when relevant (saves 30 min/week, improves open rate by 15%, etc.)

**Example**: For Word Counter:
- Use Case 1: Academic student needing to hit 5000-word essay requirement
- Use Case 2: Content marketer measuring blog post length for SEO
- Use Case 3: Translator billing by word count

### 5. Tips & Best Practices (300 words)
- 4-5 actionable tips
- Mix of pro tips (advanced users), common mistakes, and workflow integration
- Practical advice that goes beyond "just use the tool"
- Tone: experienced mentor sharing what you've learned

**Example**: For JSON Formatter:
- Tip 1: Validate early in development (catch errors before they cause production bugs)
- Tip 2: Use minify for APIs (smaller payload = faster transmission)
- Mistake: Don't assume copied JSON is valid (copy-paste errors happen)
- Workflow: Integrate with your editor shortcuts for faster formatting

### 6. FAQs (100 words)
- 3-4 common questions
- Short, direct answers (1-2 sentences per answer)
- Cover scope, compatibility, privacy, export options, offline access

**Example**:
- Q: Is my data private?
- A: Yes. All processing happens in your browser. We don't store or transmit your data.

## Keyword Strategy

- **Primary keyword**: Tool name (e.g., "JSON formatter") — use 3-5 times throughout, naturally
- **Secondary keywords**: Tool variants (e.g., "JSON beautifier", "JSON validator") — use 2-3 times each, scattered
- **Placement**: Primary keyword in intro + one subheading. Secondary keywords in Features and Examples sections.
- **NO keyword stuffing**: Should read naturally. A reader shouldn't notice the keywords.

## Writing Style

**DO**:
- Short, punchy sentences
- Active voice ("You can format JSON" not "JSON can be formatted")
- Specific examples with real data
- Conversational tone ("You might be wondering..." "Here's the thing...")
- Real benefits ("Save 10 minutes per day" not "Improve efficiency")

**DON'T**:
- Generic phrases ("In this article, we will discuss...")
- Flowery language ("magnificent tool", "revolutionize your workflow")
- Vague claims ("best tool on the market")
- Marketing fluff ("our amazing team...")
- Repetitive sentences
- Passive voice

## Final Checklist Before Submitting

- [ ] Word count: 2000-2200 words
- [ ] No plagiarism (original content)
- [ ] 3-5 sources (if statistics/claims are included, they should be cited)
- [ ] Spelling and grammar checked
- [ ] No AI-generated sounding phrases
- [ ] Primary keyword appears 3-5 times naturally
- [ ] Article flows naturally (reads like one piece, not sections stitched together)
- [ ] Real examples provided (not generic "your text here")
- [ ] Ready for 2-3 internal links to be added (todo will add these)
