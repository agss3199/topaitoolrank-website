## Security Audit Summary

### Input Validation

✅ **validateInvoiceField()**:
- Explicitly rejects: `< > & " ' backtick` via regex `/[<>"'`&]/`
- Allows: Unicode letters/numbers + safe punctuation (hyphens, underscores, periods, slashes, commas, colons, parentheses, plus, at-sign)
- Applied to ALL fields: company, client, items, notes, terms

✅ **validateInvoiceNumber()**:
- Whitelist regex: `^[a-zA-Z0-9_-]{1,50}$`
- Prevents filename injection

✅ **Validation Order**:
- All fields validated BEFORE html2canvas (prevents XSS to canvas)

### Logging & Observability

✅ **Framework Logger**:
- Zero console.error in production code (verified all 3 locations cleaned)
- Structured event keys: `pdf.generation.start`, `pdf.generation.ok`, `pdf.generation.failed`
- Error type extracted (not raw message): `error: error instanceof Error ? error.constructor.name : typeof error`

- Script tags: `<script>alert(1)</script>` rejected
- Event handlers: `onerror=alert(1)` rejected
- Quotes: Single and double quotes rejected
- Backticks: Rejected
- Ampersands: Rejected (HTML entity prefix)
- All 26 tests pass ✓

**Verdict**: ✅ **PASS** — Comprehensive security validation with tests proving injection attempts fail.
