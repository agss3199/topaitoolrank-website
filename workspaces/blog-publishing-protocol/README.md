4. **Growth Roadmap** (56 articles in 90 days; traffic projections)
5. **Keyword Research** (4-phase methodology; pre-validated keywords)
6. **Internal Linking Strategy** (pillar-to-cluster network; topical authority)

**Publish**: 3 Pillars + 13 Cluster articles (16 total)

**Timeline:**
- Days 1-9: Publish all 3 pillars

## SEO Optimization Standards

### On-Page (Every Article Must Have)

- [ ] **Title**: 50-60 characters; primary keyword at start; power word
- [ ] **Meta**: 155-160 characters; includes keyword once; CTA
- [ ] **H1**: Matches title exactly; appears once
- [ ] **H2/H3**: Keywords in 40-50% of headings; proper hierarchy
- [ ] **Word Count**: Target by article type (2,000-5,000 words)
- [ ] **Readability**: Flesch Reading Ease ≥ 60 (6th-grade level)
- [ ] **Internal Links**: 5-7 contextual links; includes pillar link
- [ ] **Images**: 1 per 500 words; ≤ 200KB; keyword-rich alt text
- [ ] **First 100 Words**: Include primary keyword
- [ ] **Active Voice**: 80%+ of sentences
- [ ] **Paragraphs**: Max 4 sentences; avg sentence ≤ 15 words

### Technical (Site Level)

- [ ] **Core Web Vitals**: LCP ≤ 2.2s, FID < 50ms, CLS < 0.05
- [ ] **Mobile Optimization**: Responsive, touch targets 48px+, no horizontal scroll
- [ ] **Schema Markup**: BlogPosting JSON-LD on every article
- [ ] **Sitemap**: Updated daily with all blog posts
- [ ] **robots.txt**: Allow all content; include sitemap location
- [ ] **Search Console**: Sitemap submitted; indexation monitored

## How to Use This Protocol (Step-by-Step)

### For Your First Article (Pillar Page)

1. **Day 1 (Morning):** 
   - Open `specs/pillar-page-outlines.md`
   - Copy outline for Pillar 1: "Custom Software Development Guide"
   - Open `specs/article-template.md`
   - Copy template structure

2. **Days 1-3 (Research):**
   - Follow `specs/blog-publishing-workflow.md` Phase 1 (Research)
   - Gather data, statistics, examples for each section

3. **Days 4-6 (Write):**
   - Follow Phase 2 (Write)
   - Draft using template structure
   - 4,500 words target for pillar page

4. **Days 7-8 (Optimize):**
   - Follow Phase 3 (Optimize)
   - Run through `specs/seo-optimization-checklist.md` (47 points)
   - Add internal links (plan to link to future cluster articles)

5. **Day 9 (Publish):**
   - Final QA
   - Publish to site
   - Update canonical URL, schema markup, Open Graph tags

6. **Days 10-13 (Promote):**
   - Submit URL to Google Search Console
   - Create backlink outreach (email 5-10 industry contacts)
   - Monitor initial impressions

### For Cluster Articles (After Pillars)

1. **Day 1:** Select article from `specs/growth-roadmap-90days.md` → Copy title + keyword
2. **Days 1-3:** Research using `specs/keyword-research-protocol.md` for validation
3. **Days 4-6:** Write using `specs/article-template.md`
4. **Days 7-8:** Optimize using `specs/seo-optimization-checklist.md`
5. **Day 9:** Publish
6. **Days 10-13:** Promote + link from pillar page

### For Ongoing Content (Months 4+)

1. **Monitor**: Check Google Search Console weekly for rankings, clicks, impressions
2. **Optimize**: Update underperforming articles (rewrite title/meta if CTR < 2%)
3. **Expand**: Add internal links when new articles publish
4. **Update**: Refresh old pillar pages with new data quarterly

## Common Mistakes to Avoid

| Mistake | Impact | Fix |
|---|---|---|
| Publishing without internal links | No internal link juice flow; articles won't rank well | Always link to pillar + 2-3 cluster articles |
| Weak titles (no power words) | Low CTR; fewer clicks despite impressions | Add power word ("Complete", "Guide", "2025") |
| No backlinks for pillars | Pillars take 6+ months to rank; slow growth | Guest post or founder mention 5-10 external links to pillars |
| Inconsistent publishing cadence | Traffic flatlines; algorithm doesn't prioritize site | Stick to schedule (1-2 per week); automation helps |
| Not monitoring rankings | Wasting time on non-ranking keywords | Check GSC weekly; rewrite titles for low-CTR articles |
| Content too thin vs competitors | Can't outrank; wrong keyword difficulty selection | Match or exceed top-3 word count + unique angle |
| Broken internal links | Poor UX; crawl inefficiency | Test all links before publishing; set reminder to audit quarterly |
| Poor mobile experience | High bounce rate; low rankings | Test on actual phone; prioritize responsive design |

## FAQ

**Q: How long until we see traffic?**
A: Week 1-2: Articles getting indexed (0 traffic). Week 3-6: First impressions show in GSC; 0-50 clicks. Week 7-12: 100-500 visits/month as articles rank. After 90 days: 15K-30K/month if on schedule.

**Q: What if we fall behind on publishing?**
A: Catch up incrementally; don't panic. Publish 3 articles next week instead of 2. One-week delays are recoverable. Ensure quality > quantity (40 excellent articles > 60 okay articles).

**Q: Should we focus on backlinks or content?**
A: Content first (weeks 1-6). Backlinks second (weeks 6-12 onward). Write 3 pillars + 10-15 clusters before starting backlink outreach. By then you have content worth linking to.

**Q: How many backlinks do we need?**
A: For pillars: 5-10 external backlinks by day 45 accelerates ranking. For clusters: Organic internal links from pillar are usually sufficient.

**Q: Can we repurpose content (blog → social → email)?**
A: Yes! Write blog article → Extract key quotes for social → Create email sequence from top 5 takeaways. One article = 10+ assets.

**Q: What if a topic isn't ranking?**
A: Likely causes: (1) Not linked from pillar, (2) Weak backlinks, (3) Content too thin vs competitors, (4) Wrong keyword difficulty selected. Fix: Add pillar link, build 1-2 backlinks, expand word count 20%, rewrite title with power word.

**Q: How often should we update articles?**
A: Quarterly (every 90 days) for pillar pages. Add new cluster links, refresh data/stats. Cluster articles: Update when new info available or if ranking drops.
