# Team Roles & Publishing Guide

## Team Structure: Who Does What

### **Role 1: Content Writer** (You or Freelancer)
**Responsibility**: Generate article content

**Tasks**:
1. Fill in ChatGPT prompt template
2. Paste into ChatGPT
3. Copy generated draft
4. Request revisions in ChatGPT (if needed)
5. Add value-adds manually:
   - Statistics with sources
   - Real company examples
   - Frameworks
   - Insert placeholder text for images: "INSERT: [image description]"
6. Hand off to Editor

**Time per article**: 6 days (Days 4-9 of 13-day cycle)

**Who this could be**:
- ✅ You (if you have writing experience)
- ✅ Your team member
- ✅ Freelance writer (Upwork, Fiverr)
- ✅ Content agency

**Estimated cost**: $0-500/article (if freelance)

---

### **Role 2: SEO Specialist** (You or Team Member)
**Responsibility**: Keyword research, optimization, monitoring

**Tasks**:

#### **Phase 1: Keyword Research (Days 1-3)**
1. Validate keyword (search volume, competition, intent)
2. Analyze top-3 competing articles
3. Identify 2-3 secondary keywords
4. Create article outline
5. Help writer understand what to focus on

**Task**: Use `specs/keyword-research-protocol.md`

#### **Phase 2: SEO Optimization (Day 8)**
1. Run 47-point SEO checklist (`specs/seo-optimization-checklist.md`)
2. Optimize title (50-60 chars, power word, keyword at start)
3. Optimize meta description (155-160 chars, benefit + CTA)
4. Verify: readability score ≥ 60 (use Grammarly)
5. Verify: keyword density correct
6. Verify: internal links (5-7) present
7. Check: images have keyword-rich alt text
8. Create: BlogPosting schema JSON-LD

**Tools needed**:
- Grammarly (free) — readability checking
- Moz Bar (free) — competitor analysis, DA metrics
- Google Keyword Planner (free) — keyword validation
- SEMrush free tier (optional)

#### **Phase 3: Post-Publish Monitoring (Day 13+)**
1. Submit to Google Search Console
2. Monitor GSC weekly (impressions, clicks, CTR)
3. Identify low-CTR articles (< 1.5%)
4. Recommend title/meta rewrites
5. Track domain authority growth (monthly)

**Time per article**: 2-3 days total
- Research: 1.5 days
- Optimization: 0.5 days
- Monitoring: 0.5 days (weekly, ongoing)

**Who this could be**:
- ✅ You (if you have SEO knowledge)
- ✅ Your marketing team member
- ✅ SEO freelancer (Upwork, PeoplePerHour)
- ✅ SEO agency ($1-5K/month)

**Estimated cost**: $0-300/month (if freelance)

---

### **Role 3: Editor/QA** (You or Team Member)
**Responsibility**: Quality control, formatting, publishing

**Tasks**:

#### **Phase 1: Content Review (Day 8)**
1. Read article for grammar/clarity
2. Check for brand voice consistency
3. Verify all facts and statistics have sources
4. Ensure examples are clear and specific
5. Request revisions from writer if needed

#### **Phase 2: Pre-Publish QA (Day 9)**
1. Run through pre-publish checklist:
   - No grammar/spelling errors
   - All links work (no 404s)
   - All image placeholders resolved
   - Mobile preview renders correctly
   - Plagiarism check (Copyscape)
2. Create/insert images in CMS
3. Set metadata (title, meta, slug, canonical URL)
4. Add schema markup (JSON-LD)
5. Publish article

#### **Phase 3: Post-Publish Verification**
1. Check article live on website
2. Verify on mobile/desktop
3. Test all internal links
4. Verify canonical URL set correctly
5. Monitor for errors (404s, broken images)

**Time per article**: 2 days
- Review: 1 day
- QA + publishing: 1 day

**Who this could be**:
- ✅ You
- ✅ Your team member
- ✅ Freelance editor (Upwork)

**Estimated cost**: $0-200/article (if freelance)

---

### **Role 4: Image Designer** (Optional; can be Writer or Tool)
**Responsibility**: Create diagrams, images, infographics

**Tasks**:
1. Create diagrams from outlines (process flows, architecture)

**Tasks**:

- Outreach: 1-2 hours/day ongoing

**Who this could be**:
- ✅ You
- ✅ Growth/marketing team member
- ✅ PR agency ($1-3K/month)
- ✅ Freelance outreach specialist ($500-1K/month)

**Estimated cost**: $500-3K/month (or DIY)

# Publishing Guide: How to Publish Your Articles

## Step 1: Choose Your Platform

### **Option A: WordPress.com or WordPress.org** (Recommended for Beginners)

**What it is**: Easiest blogging platform; drag-and-drop publishing

**Steps to publish**:
1. Log into WordPress dashboard
2. Click **"Posts"** → **"Add New"**
3. Paste article content into editor
4. Set title, slug, permalink
5. Add featured image
6. Set category/tags
7. Click **"Publish"**

**Metadata setup** (MUST DO):
- Title: Set in post editor
- Meta description: Use **"Yoast SEO"** plugin (free)
- URL slug: `/blog/keyword-separated-by-hyphens/`
- Canonical URL: Set in Yoast SEO
- Schema markup: Auto-added by Yoast SEO

**Pros**:
- ✅ Easy to use
- ✅ Built-in SEO tools (Yoast SEO plugin)
- ✅ Mobile responsive
- ✅ Good for beginners

**Cons**:
- ❌ Monthly cost ($5-15/month)
- ❌ Limited customization
- ❌ Hosting limitations

**Cost**: $5-15/month hosting + Yoast SEO Pro ($89/year optional)

**Recommended Setup**:
1. Use WordPress.org (self-hosted)
2. Install Yoast SEO plugin (free)
3. Use "Rank Math" or "All in One SEO" (free alternatives to Yoast)
4. Publish articles following template below

---

### **Option B: Webflow** (Good for Designers)

**What it is**: Visual website builder with CMS; great for custom design

**Steps to publish**:
1. Log into Webflow
2. Go to **Collections** → **Blog Posts** → **Create Item**
3. Fill in fields (title, slug, content, featured image, meta)
4. Click **"Publish"**

**Metadata setup**:
- Webflow has built-in SEO fields
- Title, slug, meta description all in form
- Schema markup available (in settings)

**Pros**:
- ✅ Beautiful design
- ✅ Good for custom layouts
- ✅ Built-in CMS

**Cons**:
- ❌ Higher cost ($12-99/month)
- ❌ Steeper learning curve
- ❌ Less SEO automation than WordPress

**Cost**: $12-99/month + designer time

---

### **Option C: Next.js (Your Website Uses This!)** (Best Control)

**What it is**: React framework; most customizable, requires coding

**Your site already uses Next.js**, so this is your advantage.

**Steps to publish** (if you want to add blog to existing Next.js site):

**Step 1: Create Blog Directory Structure**
```
your-site/
├── pages/
│   ├── blog/
│   │   ├── index.tsx (blog home)
│   │   └── [slug].tsx (individual article)
│   └── api/
│       └── blog/
│           ├── articles.json (or use database)
│           └── articles/
│               ├── article-1.md (markdown)
│               └── article-2.md
```

**Step 2: Create Article Files (Markdown)**
```markdown
---
title: "Custom Software Development Guide"
description: "Learn what custom software development is..."
image: "/images/custom-software.png"
date: "2025-05-02"
author: "Your Name"
slug: "custom-software-development-guide"
keywords: "custom software, development"
---

# Article Content Here

Your article text...
```

**Step 3: Create Dynamic Page (`pages/blog/[slug].tsx`)**
```typescript
import { getAllArticles, getArticle } from '../lib/blog'

export default function BlogPost({ article }) {
  return (
    <article>
      <h1>{article.title}</h1>
      <img src={article.image} alt={article.title} />
      <div>{article.content}</div>
    </article>
  )
}

export async function getStaticPaths() {
  const articles = getAllArticles()
  return {
    paths: articles.map(a => ({ params: { slug: a.slug } })),
    fallback: false
  }
}

export async function getStaticProps({ params }) {
  const article = getArticle(params.slug)
  return {
    props: { article },
    revalidate: 86400 // ISR: revalidate daily
  }
}
```

**Step 4: Add Schema Markup**
```typescript
import Head from 'next/head'

export default function BlogPost({ article }) {
  const schema = {
    "@context": "https://schema.org",
    "@type": "BlogPosting",
    "headline": article.title,
    "image": article.image,
    "datePublished": article.date,
    "author": { "@type": "Organization", "name": "Your Site" }
  }
  
  return (
    <>
      <Head>
        <title>{article.title}</title>
        <meta name="description" content={article.description} />
        <script type="application/ld+json">{JSON.stringify(schema)}</script>
      </Head>
      <article>{article.content}</article>
    </>
  )
}
```

**Step 5: Deploy**
```bash
git add .
git commit -m "Add blog article: [title]"
git push
# Vercel auto-deploys (if using Vercel)
```

**Pros**:
- ✅ Full control
- ✅ Super fast (Next.js optimization)
- ✅ SEO perfect (static generation)
- ✅ No monthly cost (Vercel free tier)

**Cons**:
- ❌ Requires developer (you or teammate)
- ❌ More technical setup
- ❌ No drag-drop editor

**Cost**: Free (if using Vercel + your dev time)

   ```

   │       ├── article-1.md

   ```

2. **Create `pages/blog/[slug].tsx`** (dynamic blog post page)
   - Reads markdown files
   - Renders with Next.js styling
   - Adds schema markup automatically

3. **Deploy** to Vercel (your current host)

#### **Week 2+: Publish Each Article**

**For each new article**:

1. **Create markdown file**: `content/blog/custom-software-development-guide.md`
   ```markdown
   ---

   date: "2025-05-02"
   keywords: "custom software development"
   image: "/images/article-1.png"
   ---

   # Article Content
   ...
   ```

2. **Push to GitHub**:
   ```bash
   git add content/blog/custom-software-development-guide.md
   git commit -m "docs: Add blog article - custom software development"
   git push
   ```

3. **Vercel auto-deploys** (takes 30 seconds)

4. **Submit to Google Search Console**:

   - Inspect URL: `/blog/custom-software-development-guide/`

**That's it!** Article is live and indexing.

#### **Pros of This Approach**:

- ✅ Perfect SEO (static generation, ISR)
- ✅ No database setup
- ✅ No monthly costs

#### **Cons**:
