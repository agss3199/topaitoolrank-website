# 06-blog-sitemap-seo

**Implements**: specs/blog-indexing.md § 5. SEO & Canonical Links (Sitemap updates)  
**Depends On**: 01-blog-tag-pages, 02-blog-category-pages (routes must exist before sitemap includes them)  
**Capacity**: ~80 LOC load-bearing logic / 3 invariants (tag pages at 0.8 priority, category pages at 0.8 priority, individual posts at 0.9 priority) / 2 call-graph hops (`getAllPosts` → enumerate tags/categories → append to sitemap entries) / Updates the existing sitemap to include all generated tag index and category index pages with correct priorities and lastmod dates.  
**Status**: ACTIVE

## Context

The existing `app/sitemap.ts` generates sitemaps for individual posts and static pages. After todos 01 and 02 create tag and category index pages, those pages must be added to the sitemap for search engine discovery.

This todo is the final piece of the blog scalability track. All blog index pages (tags, categories) will be discoverable by crawlers after this lands.

## Scope

**DO:**
- Modify `app/sitemap.ts` to enumerate all unique tags and add `/blogs/tag/[slug]` entries with:
  - `priority: 0.8`
  - `lastmod`: the maximum `updatedAt` (or `publishedAt` if no updatedAt) among posts with that tag
  - `changefreq: 'weekly'`
- Modify `app/sitemap.ts` to enumerate all unique categories (from published posts) and add `/blogs/category/[category]` entries with:
  - `priority: 0.8`
  - `changefreq: 'weekly'`
- Verify individual post entries use `priority: 0.9` (update if they currently differ)
- Verify `app/layout.tsx` or the existing `<head>` structure already has the RSS alternate link from todo 03 (do not duplicate it)

**DO NOT:**
- Change priorities for non-blog pages (homepage, tool pages)
- Add draft post URLs to the sitemap
- Add `/blogs/category/[category]` entries for categories with zero published posts (avoids sitemap pointing to effectively-empty pages)
- Rebuild the sitemap from scratch — extend the existing `app/sitemap.ts`

## Deliverables

**Modify:**
- `app/sitemap.ts` — add tag and category page entries

**Tests:**
- `__tests__/blog-sitemap.test.ts` — unit tests for sitemap entry generation

## Testing

**Unit tests (Tier 1):**
- `test_sitemap_includes_tag_pages` — sitemap entries include `/blogs/tag/ai-tools` when a published post has tag "AI Tools"
- `test_sitemap_excludes_draft_post_tags` — tag appearing only on draft posts is NOT in the sitemap
- `test_sitemap_includes_category_pages` — sitemap includes `/blogs/category/AI%20Tools` entry
- `test_sitemap_tag_page_priority_is_0_8` — all tag page entries have priority `0.8`
- `test_sitemap_category_page_priority_is_0_8` — all category page entries have priority `0.8`
- `test_sitemap_individual_posts_priority_is_0_9` — individual post entries have priority `0.9`
- `test_sitemap_no_category_for_zero_post_categories` — category entry not generated if no published posts belong to it

**Manual checks:**
- Run `next build`; visit `/sitemap.xml` in the running server
- Confirm `/blogs/tag/ai-tools` (or similar) appears in the sitemap XML
- Confirm `/blogs/category/AI%20Tools` appears in the sitemap XML
- Confirm draft post slugs do NOT appear

## Implementation Notes

- URL encoding: category names with spaces (like "AI Tools") must be percent-encoded in the sitemap URL: `/blogs/category/AI%20Tools`. Use `encodeURIComponent(category)`.
- Tag slugs in the sitemap must use `slugifyTag()` from todo 01 — they must match the routes generated by `generateStaticParams` exactly.
- `lastmod` for tag/category pages: use `Math.max(...posts.map(p => new Date(p.frontmatter.publishedAt).getTime()))` then format as ISO date string. If no posts (edge case filtered by DO NOT above), skip the entry entirely.
- The existing sitemap may already include individual post entries — do not remove or modify those; only append the new tag/category entries.
