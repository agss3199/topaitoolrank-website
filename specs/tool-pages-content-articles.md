# Tool Pages Content Articles Specification

## Scope

2000-word articles for 9 micro-SaaS tool pages (excluding WA-Sender). Articles positioned below the interactive tool and above the footer. Written by humans (high-quality, non-AI), optimized for SEO without penalty risk, focused on user value first.

## Article Structure (2000 words per tool)

### 1. Introduction (200 words)

**Purpose**: Hook reader, establish relevance, explain what the tool is and why they need it.

**Content**:
- Engaging opening sentence that resonates with the tool's audience
- Clear problem statement ("Many people struggle to...", "One of the biggest challenges...")
- Brief explanation of what the tool does and its core benefit
- Why this specific problem matters to the reader
- Quick benefit summary ("This tool helps you...")
- Call to action ("Let's dive into how to use it")

**Tone**: Conversational, empathetic, problem-focused. Not promotional. Reader-first.

**Keywords**: Naturally incorporate 1-2 primary keywords (e.g., "JSON formatter", "word counter"). No keyword stuffing.

### 2. How to Use (500 words)

**Purpose**: Step-by-step walkthrough, make the tool immediately usable.

**Content**:
- Step 1: Getting started / Accessing the tool
- Step 2: Basic input (what to enter, format expectations)
- Step 3: Using key features (what buttons to click, options to choose)
- Step 4: Interpreting results (what the output means)
- Step 5: Tips for common use cases

**Format**: Numbered steps, clear language, avoid jargon. Use concrete examples with real data the reader recognizes.

**Example** (JSON Formatter):
```
Step 1: Paste your JSON into the input box on the left
Step 2: Click "Format" to beautify and validate
Step 3: Check the results on the right panel
Step 4: Use the "Copy" button to copy formatted JSON
Step 5: Optional: Use "Minify" for production deployment
```

**Tone**: Instructional, encouraging, supportive. Assume the reader is new to this tool.

**Keywords**: 2-3 secondary keywords naturally woven in (e.g., "JSON validation", "JSON beautifier").

### 3. Features & Benefits (400 words)

**Purpose**: Explain why this tool is valuable, differentiate it, show broader use cases.

**Content**:
- Key feature 1: What it does + why it matters
- Key feature 2: What it does + why it matters
- Key feature 3: What it does + why it matters
- Benefits for different user types (e.g., for developers, for content creators, for marketers)
- Comparison to manual alternatives (without naming competitors)
- Why this tool saves time/money/effort

**Format**: Feature-benefit pairs. Not just listing features, but explaining impact.

**Example** (Email Subject Tester):
```
Feature: A/B test preview
Benefit: See how your subject line appears in different email clients before you send it

Feature: Character count and preview width
Benefit: Avoid cut-off text on mobile devices where 30% of emails are opened

Feature: Spam word detection
Benefit: Avoid spam filters so your email reaches the inbox instead of junk
```

**Tone**: Educational, benefits-focused, reader-centric. Answer "What's in it for me?"

### 4. Examples & Use Cases (500 words)

**Purpose**: Show real-world applications, demonstrate value in context, inspire reader action.

**Content**:
- Example 1: Concrete scenario with input/output (developer use case)
- Example 2: Concrete scenario (marketer/creator use case)
- Example 3: Concrete scenario (business/enterprise use case)
- Each example shows: what the reader was trying to do, how they used the tool, what they achieved
- Brief explanation of why this example matters

**Format**: Narrative storytelling. Paint a picture. Make it relatable.

**Example** (Invoice Generator):
```
Use Case 1: Freelance Writer
Sarah needs to invoice a client for 10 blog articles at $200 each.
Instead of manually creating an invoice in Excel (error-prone, looks unprofessional),
she enters her details and line items into the Invoice Generator,
gets a professional PDF in seconds, and emails it the same day.
Result: Looks professional, client pays faster (professional invoices improve payment speed by 10-15%).

Use Case 2: Small Business Owner
Marcus runs a digital marketing agency. He invoices 8-12 clients per month.
Creating invoices manually takes 15+ minutes per invoice = 2+ hours per month.
With the Invoice Generator: 1 minute per invoice = 12-15 minutes per month.
Result: Saves ~100 hours per year, focuses on actual client work instead of admin.
```

**Tone**: Real, practical, outcome-focused. "Here's what real people do with this."

### 5. Tips & Best Practices (300 words)

**Purpose**: Help reader get even more value, optimize their use, avoid common mistakes.

**Content**:
- Tip 1: Pro tip for better results
- Tip 2: Common mistake to avoid
- Tip 3: Advanced technique for power users
- Tip 4: How to integrate this tool into your workflow
- General best practice summary

**Format**: Concise bullets with brief explanations. Actionable advice.

**Example** (JSON Formatter):
```
Pro Tip 1: Use minify mode for APIs
Minified JSON is smaller and faster to transmit. Use this for production APIs.

Common Mistake: Assuming copied JSON is valid
Always validate first. The formatter will catch errors like missing commas.

Advanced: Combine with editor shortcuts
Most code editors (VS Code, etc.) have built-in JSON formatting.
The online tool is great for quick checks without opening an editor.

Best Practice: Validate early in development
Catch JSON errors early before they cause runtime bugs in production.
```

**Tone**: Experienced mentor. "Here's what I've learned about using this tool."

### 6. FAQs (100 words)

**Purpose**: Address common questions, reduce friction, improve user confidence.

**Content**:
- FAQ 1: "Can I use this for...?" (scope/compatibility)
- FAQ 2: "Is my data private?" (privacy/security)
- FAQ 3: "Can I download results?" (export options)
- FAQ 4: "Does this work offline?" (technical requirement)

**Format**: Q & A, short answers (1-2 sentences per answer).

**Example**:
```
Q: Is my data private?
A: Yes. All processing happens in your browser. We don't store or transmit your data.

Q: Can I use this on my phone?
A: Yes. The tool works on all devices with a web browser.

Q: Can I export the formatted JSON?
A: Yes. Use the "Copy" button to copy formatted JSON, or save it locally.
```

**Tone**: Clear, reassuring, directly answers the question.

## Content Quality Standards

### MUST: Human-Written, Non-AI

- Articles written by humans with domain knowledge
- Not AI-generated content passed off as human writing
- Authentic voice and perspective
- Verifiable examples grounded in real use cases

**Why**: AI-generated content has detectable patterns (repetitive phrasing, generic examples, formulaic structure) that Google's systems identify. Content written by humans reads naturally and builds reader trust.

### MUST: Valuable & Genuinely Useful

- Primary focus: Help the reader solve a problem or achieve a goal
- Not promotional or sales-focused
- Avoids misleading claims about the tool's capabilities
- Examples are realistic and achievable for typical users
- Tone is educational, not persuasive

**Why**: Users can detect low-value content (tool reviews, keyword-stuffed articles). Quality content keeps users on the page longer, reduces bounce rate, and signals to search engines that the page is authoritative.

### MUST: SEO-Optimized Without Penalty Risk

**Keyword strategy**:
- 1-2 primary keywords naturally woven into introduction + How to Use
- 2-3 secondary keywords in Features & Examples sections
- No keyword stuffing, no unnatural repetition
- Keywords appear in headings (H2/H3 level)
- NO all-caps keywords or keyword overload in any section

**Meta elements** (separate from article):
- Meta title: 50-60 chars, includes primary keyword, brand name
- Meta description: 150-160 chars, includes primary keyword, value proposition
- H1 tag (article title): Tool name + primary benefit (e.g., "JSON Formatter: Beautify and Validate JSON Online")

**Link strategy** (cross-linking covered in separate spec):
- 2-3 internal links to related tools (naturally placed)
- 1 link back to homepage or tools listing
- External links only to trusted sources (optional)

**Content depth**:
- 2000+ words (this signals authority and comprehensiveness to search engines)
- Well-structured with clear headings (H2/H3 hierarchy)
- Multiple paragraphs (no walls of text)
- Uses lists and bullet points for scannability

**Technical SEO**:
- Mobile-responsive layout (covered in header/footer spec)
- Fast load time (tools already optimized, article content is text-only)
- Proper heading hierarchy (H1 > H2 > H3, no skipping levels)

**No Black-Hat Tactics**:
- NO hidden text (white text on white background)
- NO keyword cloaking (different content for users vs search engines)
- NO duplicate content across tool pages (each article unique)
- NO thin content reused across multiple pages
- NO automated content generation or spinning

### MUST: Well-Structured & Readable

- Clear section headings (H2 level)
- Short paragraphs (3-4 sentences max)
- Active voice preferred
- Subheadings within long sections for scannability
- Bullet points for lists
- Emphasis (bold/italic) on key concepts

**Readability metrics**:
- Flesch Reading Ease score: 60-70 (conversational, accessible to general audience)
- No sentences longer than 20 words on average
- Avoid jargon without explanation

### MUST: Authentic Examples & Data

- Real use cases based on actual user workflows
- If numbers/statistics are cited, they must be from credible sources (cite the source)
- Examples walkthrough step-by-step (don't skip intermediate steps)
- Avoid made-up scenarios; use realistic situations

**Why**: Fake examples and unverified statistics undermine credibility. Readers (and Google) value authentic, grounded content.
