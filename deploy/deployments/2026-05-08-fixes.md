## Issues Fixed

### 1. Animation Not Working on Desktop
**Problem**: Neural core rings animated on mobile but not on desktop (1024px+).

**Root Cause**: Animations were defined in base styles, but heavy reliance on CSS cascade media queries caused uncertainty. Added explicit desktop media query for clarity.

**Fix**: 
- Added `@media (min-width: 1024px)` with explicit animation declarations for all rings
- Ensures `ring-one`, `ring-two`, `ring-three` have proper animation properties on desktop
- Animation timing: rotateRing 16s/10s, rotateRingReverse 13s

**Verification**: Animations now visible on desktop (1024px+)

### 2. Header Covering Page Content
**Problem**: Fixed header with `position: fixed; z-index: 1000;` was overlaying page content without vertical offset.

**Root Cause**: Hero section had `padding: 150px 0 90px;` which didn't account for the fixed header positioned at `top: 18px` with `min-height: 72px;` (~90px total height).

**Fix**:
- Increased hero section padding-top from 150px to 230px
- This creates sufficient space for fixed header (~90px) + additional buffer
- Header no longer hides content at top of page

**Verification**: Hero section content now visible below fixed header
